📋 Mockup Notes: Leaderboard — Exact Scores & Correct Results

  • Two new columns added:
    • "Exact" — Count of matches where the player predicted the exact final score correctly.
    • "Results" — Count of matches where the player predicted the correct outcome (home win / draw / away win) but NOT the exact score. This explicitly excludes exact score matches to avoid double-counting.
  • Both columns are sortable — clicking the header toggles ascending/descending sort, same as existing columns.
  • Default sort order: The table sorts by Total Points (desc) first, then Exact Scores (desc) as a tiebreaker, then Correct Results (desc) as a second tiebreaker. This is reflected in rank assignment.
  • Data source: The MatchScore model already stores correctExactScore (Boolean) and correctResult (Boolean) fields. The counts can be derived by summing these per player from scored matches only.
  • "Correct Results" logic: Only counts matches where correctResult === true AND correctExactScore === false. This ensures no overlap with the Exact column.
  • Column header tooltips: Hovering over any column header shows a styled tooltip popup explaining what that column means. Uses Tailwind group-hover with named groups for a dark tooltip that appears below the header. This gives more control over styling and readability than the browser's default title tooltip. Left-aligned columns show the tooltip anchored left; right-aligned columns anchor right.

🔄 Sort Behaviour

  • Clicking any column header sorts by that column (toggling asc/desc).
  • Default load order: Total (desc) → Exact (desc) → Results (desc).
  • Active sort column shows ↑ or ↓; inactive columns show ↕.
